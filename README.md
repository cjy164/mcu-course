实物如图

小组完成以STM32的智能手表

可实现时间的显示，记录步数和传感器测试的数据

所用物料包括一块1.3寸OLED显示屏，830孔的面包板，MPU-6050模块

系统方案设计：1. 系统总体架构

采用主控核心层→外设硬件层→STM32固件层→蓝牙通信层→手机APP应用层四层架构：

\- 核心层：STM32F4主控统筹所有外设、数据运算、任务调度

\- 硬件层：显示、传感、电源、振动、按键等外设硬件

\- 固件层：基于FreeRTOS实现多任务管理、外设驱动、算法处理

\- 应用层：通过蓝牙和安卓/IOS手机APP双向同步数据、下发设置指令。

软件流程图

1. 上电初始化：外设驱动初始化→FreeRTOS任务创建→RTC时钟校准→传感器自检

2. 循环主逻辑：

\- 待机模式：屏幕低功耗休眠，RTC维持走时

\- 按键中断任务：唤醒屏幕、切换功能页面

\- 蓝牙通信任务：接收APP指令、上传生理/运动数据

\- 提醒任务：来电/消息触发马达振动提醒

3. 异常处理：低电量自动降频、传感器故障报错弹窗

4. 关机/深度休眠：长按按键触发低功耗休眠

硬件部分

1\. 核心电路原理图关键设计

1. 最小系统电路：STM32F407配置8MHz外部晶振，系统主频超频至168MHz；配套复位电路、BOOT启动配置电路；3.3V供电去耦电容就近摆放。

2. 电源电路：Type-C接入TP4056完成锂电池充电，充电电流设置500mA；锂电池输出经AMS1117稳定3.3V给MCU和外设供电，预留电压检测分压电路，MCU ADC采集电池电量。

3. 传感电路：MPU6050、MAX30102通过I2C总线挂载，上拉4.7kΩ电阻；MAX30102增加光源限流电阻，降低光源功耗提升续航。

4. 显示与交互电路：TFT屏用SPI高速通信，触摸芯片I2C接口；2个轻触按键做上下电/页面切换中断触发；马达用三极管MOS管驱动，GPIO控制通断。

5. 蓝牙电路：HC-05串口和STM32 USART引脚交叉连接，配套分压电路匹配3.3V电平，避免电平烧坏芯片。
