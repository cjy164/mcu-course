# STM32CubeMX + STM32CubeIDE 配置与烧录指南

---

## 第一步：安装软件

### 1.1 下载安装 STM32CubeMX

1. 访问 ST官网: https://www.st.com/en/development-tools/stm32cubemx.html
2. 点击 **Get Software** → 注册/登录 → 下载安装包
3. 运行安装程序，一路 Next 即可

### 1.2 下载安装 STM32CubeIDE

1. 访问: https://www.st.com/en/development-tools/stm32cubeide.html
2. 点击 **Get Software** → 下载对应版本
3. 安装时勾选 **STM32F1xx** 系列固件包

### 1.3 安装 ST-LINK 驱动（如果没有）

连接 ST-LINK 调试器后，Windows 会自动安装驱动。如需手动安装：
- 访问: https://www.st.com/en/development-tools/stsw-link009.html
- 下载安装 **ST-LINK USB driver**

## 第二步：用 STM32CubeMX 生成基础工程

### 2.1 新建工程

1. 打开 **STM32CubeMX**
2. 点击 **File → New Project**
3. 在 **Part Number Search** 输入 `STM32F103C8`
4. 选中 **STM32F103C8Tx** → 点击 **Start Project**

### 2.2 配置系统时钟 (RCC)

- 点击 Pinout & Configuration 标签
- 左侧 **System Core → RCC**
- **High Speed Clock (HSE)** → 选择 **Crystal/Ceramic Resonator**
- **Low Speed Clock (LSE)** → 选择 **Disabled**

### 2.3 配置调试接口 (SYS)

- **System Core → SYS**
- **Debug** → 选择 **Serial Wire**（使用 SWD 调试时需要）
- **Timebase Source** → 选择 **TIM1**（避免与 FreeRTOS 冲突）

### 2.4 配置 I2C1

- **Connectivity → I2C1**
- **Mode** → 选择 **I2C**
- 确认引脚：**PB6 = I2C1_SCL**, **PB7 = I2C1_SDA**
- **Parameter Settings**:
  - Speed Mode: **Standard (100KHz)**

### 2.5 配置 USART1（可选，拓展蓝牙时用）

- **Connectivity → USART1**
- **Mode** → 选择 **Asynchronous**
- 确认引脚：**PA9 = USART1_TX**, **PA10 = USART1_RX**
- **Parameter Settings** → Baud Rate: **9600**

### 2.6 配置 GPIO

- 点击 **PA0** 引脚，选择 **GPIO_EXTI0**
- 点击 **PA2** 引脚，选择 **GPIO_Output**
- 左侧 **System Core → GPIO**
  - **PA0**: GPIO mode = **External Interrupt Mode with Falling edge trigger detection**
           GPIO Pull-up/Pull-down = **Pull-up**
  - **PA2**: Output level = **Low**

### 2.7 配置 TIM2（1秒定时器）

- **Timers → TIM2**
- **Clock Source** → 选择 **Internal Clock**
- **Parameter Settings**:
  - Prescaler: **7200-1**
  - Counter Period: **10000-1**
  - Auto-reload preload: **Enable**
- **NVIC Settings** → 勾选 **TIM2 global interrupt**

### 2.8 配置 NVIC 中断优先级

- **System Core → NVIC**
  - **EXTI line0 interrupt**: 勾选 Enabled, Preemption Priority = **5**, Sub Priority = **0**
  - **TIM2 global interrupt**: 勾选 Enabled, Preemption Priority = **5**, Sub Priority = **0**

### 2.9 配置 FreeRTOS

- **Middleware → FREERTOS**
- **Interface** → 选择 **CMSIS_V2**
- **Tasks and Queues** 标签:
  - 删除默认的 `defaultTask`
  - 点击 **Add** 创建以下任务（或在代码中用 xTaskCreate 创建，本项目已实现，可跳过）：

| Task Name | Stack Size (Words) | Priority | Entry Function | Parameter |
|:---------:|:------------------:|:--------:|:--------------:|:---------:|
| oledTask  | 256 | Normal | OLED_Task | (不填) |
| sensorTask | 256 | AboveNormal | Sensor_Task | (不填) |
| buttonTask | 256 | High | Button_Task | (不填) |

> **建议**: 不在 CubeMX 中创建任务，直接使用本项目 `main.c` 中的 `xTaskCreate()` 代码，更灵活。

- **Config Parameters** 标签:
  - TOTAL_HEAP_SIZE: **14336** (14KB)
  - 其余保持默认

### 2.10 配置时钟 (Clock Configuration)

点击 **Clock Configuration** 标签，按以下设置：

```
HSE (8MHz) → PLLSRC → PLLMUL(x9) → SYSCLK(72MHz)
                                    → AHB Prescaler(/1) → HCLK(72MHz)
                                                          → APB1 Prescaler(/2) → APB1(36MHz)
                                                          → APB2 Prescaler(/1) → APB2(72MHz)
```

- **PLL Source**: HSE
- **PLL Mul**: x9
- **System Clock**: 72MHz
- **APB1 Prescaler**: /2 (36MHz)
- **APB2 Prescaler**: /1 (72MHz)
- **Flash Latency**: 2

点击 **Resolve Clock Conflicts** 自动修正。

### 2.11 生成代码

1. 点击 **Project Manager** 标签
2. **Project Name**: `SmartWatch`
3. **Project Location**: 选择一个文件夹（如桌面）
4. **Toolchain / IDE**: **STM32CubeIDE**
5. 勾选 **Generate peripheral initialization as a pair of .c/.h files per peripheral**
6. 点击右上角 **GENERATE CODE**
7. 弹窗选择 **Open Project** → 自动打开 STM32CubeIDE

## 第三步：在 STM32CubeIDE 中整合项目代码

### 3.1 替换源文件

CubeMX 已生成基础工程，现在将本项目代码覆盖进去。

**替换 main.c**:

1. 在 STM32CubeIDE 左侧 **Project Explorer** 中
2. 展开 `SmartWatch → Core → Src`
3. 双击 `main.c` 打开
4. **全选** (Ctrl+A) → **删除**，然后将本项目 `main.c` 的 **全部内容粘贴进去**
   > 注意：保留 CubeMX 生成的 `MX_*_Init()` 函数部分，本项目代码已兼容

**添加新文件**:

1. 在 `Core/Inc` 文件夹上右键 → **New → File**
2. 依次创建以下头文件（复制本项目内容粘贴）：
   - `oled.h`
   - `mpu6050.h`
   - `soft_timer.h`
   - `ui.h`
   - `i2c.h` (替换已存在的)

3. 在 `Core/Src` 文件夹上右键 → **New → File**
4. 依次创建以下源文件：
   - `oled.c`
   - `mpu6050.c`
   - `soft_timer.c`
   - `ui.c`
   - `i2c.c` (替换已存在的)

**替换 FreeRTOSConfig.h**:

1. 在工程中找到 `FreeRTOSConfig.h`（可能在 `Core/Inc` 或 `Middlewares` 下）
2. 用本项目 `FreeRTOSConfig.h` 的内容替换

### 3.2 快速替代方案：直接导入

如果文件太多手动创建麻烦，可以这样：

1. 在 CubeMX 生成工程后，**关闭 STM32CubeIDE**
2. 用文件管理器打开生成的工程文件夹
3. 将本项目 `Core/Src/` 和 `Core/Inc/` 中的文件**直接复制**进去覆盖
4. 重新打开 STM32CubeIDE，它会自动识别新文件

### 3.3 编译验证

1. 在 STM32CubeIDE 中，点击 **Project → Build All** (或按 Ctrl+B)
2. 观察 Console 输出，应为 `Build Finished`，无错误
3. 如出现 `undefined reference` 错误，说明有文件未添加：
   - 在 `Core/Src` 文件夹上右键 → **Refresh** (F5)
   - 或者手动将 .c 文件添加到编译组

## 第四步：烧录到 STM32 芯片

### 4.1 硬件连接

```
STM32F103C8T6 最小系统板 (Blue Pill)    ST-LINK V2 调试器
┌──────────────────┐                  ┌──────────────┐
│ SWDIO (PA13/DIO) ├──────────────────┤ SWDIO        │
│ SWCLK (PA14/CLK) ├──────────────────┤ SWCLK        │
│ GND              ├──────────────────┤ GND          │
│ 3.3V (VCC)       ├──────────────────┤ 3.3V         │
└──────────────────┘                  └──────────────┘
```

同时连接好外设：

```
STM32F103  ─── PB6 ── OLED SCL ── MPU6050 SCL
           ─── PB7 ── OLED SDA ── MPU6050 SDA
           ─── 3.3V ── OLED VCC ── MPU6050 VCC
           ─── GND  ── OLED GND ── MPU6050 GND
           ─── PA0  ── 按键 (另一脚接GND)
```

> ⚠️ I2C 总线 SCL/SDA 各需 4.7kΩ 上拉到 3.3V（某些模块已含）

### 4.2 烧录步骤

1. 将 **ST-LINK V2** 插入电脑 USB 口
2. 将 **ST-LINK** 与 **STM32 最小系统板** 用杜邦线连接（见上表）
3. 给 STM32 板供电（可由 ST-LINK 的 3.3V 供电，或外接 USB）
4. 在 STM32CubeIDE 中：
   - 点击 **Run → Run** (或按 Ctrl+F11)
   - 选择 **STM32 Cortex-M C/C++ Application**
   - 点击 **OK**
5. CubeIDE 会自动编译 → 下载 → 复位运行

### 4.3 首次运行验证

烧录成功后：

1. OLED 应显示 `Starting...` 字样
2. 接着显示 `MPU6050 OK` 或 `MPU6050 ERR`
3. 进入**时间页面**（显示时:分:秒 + 日期）
4. 按下 **PA0 按键** → 切换到**传感器页面**（显示 AX/AY/AZ + 温度）
5. 再按一次 → 切回时间页面

### 4.4 常见问题

| 问题 | 解决方法 |
|:----|:---------|
| 烧录时报 `No ST-LINK detected` | 检查 ST-LINK USB 连接；检查 SWDIO/SWCLK/GND 接线 |
| `Target No Device Found` | 检查 STM32 板是否供电；检查接线 |
| OLED 不亮/不显示 | 检查 VCC/GND 接线；检查 I2C 上拉电阻 |
| `MPU6050 ERR` | 检查 MPU6050 接线；确认 I2C 地址是 0x68 |
| 编译报 `undefined reference` | 检查是否所有 .c 文件都已添加到工程 |
| 按键无反应 | 检查 PA0 接线；确认按键另一端接 GND |
